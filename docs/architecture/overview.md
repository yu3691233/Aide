# 系统概述

> 本文档介绍 AideLink 的定位、核心价值和技术选型。

---

## 系统定位

AideLink 是一个 **PC 端 AI 副驾桥接工具**，核心价值：

- **手机 ↔ PC 双向联动**：通过 ADB + Flask 桥接 Android 手机和 PC
- **AI 协作**：将手机指令转发给 PC 上的 AI 助手执行
- **桌面增强**：桌面吉祥物、系统托盘、IDE 控制

---

## 核心价值

### 1. 无缝连接

- 手机和 PC 通过 ADB 无线连接
- 自动发现和连接设备
- 支持多设备同时连接

### 2. AI 协作

- 将手机指令转发给 AI 助手
- 支持多种 AI 模型
- 任务自动拆分和委派

### 3. 桌面增强

- 桌面吉祥物
- 系统托盘快捷操作
- IDE 快速启动和控制

### 4. 自演化

- 失败记忆和学习
- 工作绕过知识库
- 持续改进和优化

---

## 技术选型

### 为什么选择 Flask？

**原因**：
1. **简单轻量**：Flask 是 Python 最轻量的 Web 框架
2. **快速开发**：适合快速原型开发
3. **扩展性强**：通过 Blueprint 支持模块化
4. **社区活跃**：丰富的扩展和文档

**权衡**：
- 性能不如 FastAPI
- 异步支持较弱
- 但对于本地网络应用足够

### 为什么选择 ADB？

**原因**：
1. **原生支持**：Android 原生调试工具
2. **功能丰富**：支持文件传输、命令执行、端口转发
3. **稳定可靠**：经过多年验证
4. **无需额外依赖**：Android 设备自带

**权衡**：
- 需要开启开发者选项
- 无线连接不如 WebSocket 稳定
- 但对于本地网络应用足够

### 为什么选择 Kotlin + Compose？

**原因**：
1. **现代语言**：Kotlin 是 Android 官方推荐语言
2. **声明式 UI**：Compose 简化 UI 开发
3. **性能优秀**：编译为原生代码
4. **类型安全**：减少运行时错误

**权衡**：
- 学习曲线较陡
- 生态不如 Java 成熟
- 但对于新项目足够

### 为什么选择 MiniMax API？

**原因**：
1. **中文优化**：对中文支持更好
2. **性价比高**：价格合理
3. **响应快速**：延迟较低
4. **功能丰富**：支持 Tool Calling

**权衡**：
- 不如 OpenAI 知名
- 文档相对较少
- 但对于中文场景足够

## 设计原则

### 1. 简单优先

- 优先选择简单易懂的方案
- 避免过度设计
- 代码清晰易读

### 2. 渐进增强

- 先实现核心功能
- 逐步添加高级功能
- 保持向后兼容

### 3. 模块化

- 组件职责单一
- 接口清晰明确
- 便于测试和维护

### 4. 安全第一

- 环境变量管理敏感信息
- 输入验证和过滤
- 最小权限原则
